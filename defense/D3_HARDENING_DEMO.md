# D3 Web Hardening Demo Guide

This guide is for the final D3 validation demo after implementing:

- parameterized SQL queries
- bcrypt password hashing
- CSRF protection with Flask-WTF
- TOTP-based MFA using `pyotp`

## 1. Rebuild the web service

After changing dependencies and login logic, rebuild the `web` container:

```bash
docker compose up --build -d web
```

If you want a clean restart of the whole stack:

```bash
docker compose down
docker compose up --build -d
```

Open the panel at:

```text
http://localhost:5001
```

## 2. Run the quick verification helper

Use the helper script to automatically check the most important D3 outcomes:

```bash
chmod +x attacks/verify_web_hardening.sh
./attacks/verify_web_hardening.sh
```

What it checks:

- `admin' --` no longer bypasses login
- password-only login is redirected to `/verify-totp`
- `/command` rejects requests without a CSRF token
- the `admin` password is stored as a bcrypt hash in SQLite

## 5. Report-ready summary paragraph

You can adapt this directly for the D3 section of the report:

> The SmartNest web panel was hardened by replacing string-built SQL queries with parameterized database access, eliminating the original login SQL injection vector. Passwords are no longer stored in plaintext; instead, they are hashed using bcrypt before persistence in SQLite. Cross-site request forgery protection was introduced using Flask-WTF, which forces every sensitive POST request to carry a valid CSRF token and blocks forged requests from external pages. Finally, time-based one-time passwords were added as a second authentication factor. During validation, the previous `admin' --` bypass failed, `sqlmap` did not identify an injectable parameter in `/login`, the CSRF proof-of-concept was rejected, and successful login required a valid six-digit TOTP code.

## 6. Troubleshooting

If the login page does not show the MFA flow you expect:

- rebuild the `web` container again
- clear your browser cookies for `localhost:5001`
- remove the old stack with `docker compose down`
- restart the stack so the updated Python dependencies install correctly

If you want to reset the TOTP enrollment state for a clean demo:

```bash
docker exec -it smartnest-web python -c "import sqlite3; conn=sqlite3.connect('/tmp/smartnest.db'); conn.execute(\"UPDATE users SET totp_secret=NULL WHERE username='admin'\"); conn.commit(); conn.close()"
```
